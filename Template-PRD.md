### Formatting Notes

This PRD template follows the Upgrade Assist document style:

- **Overview table** at top with Problem Statement, Objective, Review status, and Slack channel — no header row, labels in left column
- **Narrative** as a single vivid paragraph (scenario → friction → solution → outcome)
- **Why** section with 3–4 bold-labeled value levers
- **Success Metrics** split into Business, User-Centric, and Technical tables (5 columns: Metric, Target, Measurement, Tooling, Cadence)
- **Non-Goals** explicitly listed to prevent scope creep
- **Requirements table** with #, Name, User Story, Description, Notes columns
- **Appendices** pointing to companion docs
- **Open Questions** as bold-labeled items with specific, actionable questions

When generating the docx version, the document uses:
- Color palette: Green (#059669), Dark (#1F2937), Medium (#4B5563), Light Green (#ECFDF5)
- Font: Arial throughout
- Headers/Footers with page numbers and "Confidential — Upgrade, Inc." branding
